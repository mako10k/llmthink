# ADR-0001: エンジンを思考監査エンジンとして定義する

## Status

accepted

## Date

2026-04-30

## Context

- 本プロジェクトの目的は、意味論の厳密性よりも思考の記述明確化を優先することにある
- 記述方法の曖昧性は不許可だが、内容の暫定性や意味解釈の揺れは許容する
- 非論理的思考を抑制するため、DSL だけでなく検査系の存在が必要である
- 元の構想では検索、意味近傍、グラフ探索が前面に出ていたが、主目的とずれやすい
- 明確な矛盾、矛盾候補、DSL 契約違反、MECE 記述不備を扱う責務が必要である

## Decision

本プロジェクトのエンジンは、推論代行エンジンではなく思考監査エンジンとして定義する。

エンジンの中心責務は次のとおりとする。

- 明確な矛盾の検出
- 矛盾候補の列挙
- DSL 契約違反の検出
- MECE 性の記述的正確さの評価
- 補助的な意味論的距離や関連性の提示
- query が与えられた場合の補助的 query_result の返却

意味近傍、グラフ探索、時系列スコアは主機能ではなく補助機能として扱う。

## Alternatives Considered

- 検索エンジンを中心に据える
  - 類似検索や関係探索は可能になるが、思考の監査より検索体験が主役になりやすいため不採用
- 推論代行エンジンとして設計する
  - エンジンが正しさを生成する方向へ寄り、思考過程の外部化と再読性の向上という目的から外れやすいため不採用
- 単純なパーサとレンダラに留める
  - 記述規律の強制力が弱く、非論理的思考の抑制に十分でないため不採用

## Consequences

- 監査結果モデルと重大度モデルが必要になる
- query は主機能ではなく補助出力になる
- 意味論的距離は hint として扱われる
- エンジンの評価基準は真理判定ではなく内部整合性に寄る
- 将来、推論補助を追加する場合も監査責務を侵食しない設計が必要になる

## Auditability Notes

- 監査結果カテゴリを追加または統合する場合に再評価する
- query を主機能へ昇格させる提案が出た場合に再評価する
- ベクトル検索やグラフ探索が中心ユースケースを支配し始めた場合に再評価する
- docs/specs/requirements.md の主目的定義が変わった場合に再評価する
