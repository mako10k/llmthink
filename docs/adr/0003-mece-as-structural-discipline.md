# ADR-0003: MECE を真理判定ではなく記述規律として扱う

## Status
accepted

## Date
2026-04-30

## Context

- 本プロジェクトの主目的は思考の記述明確化である
- エンティティの意味解釈や意味論の厳密性は主目的ではない
- MECE を意味の正しさや客観的真理の証明に使おうとすると、仕様が重くなる
- 一方で、分類軸の混線や Others の乱用を防ぐためには、MECE 的な規律は有効である

## Decision

本プロジェクトでは、MECE を真理判定手段としてではなく、記述規律および監査規則として扱う。

採用する立場は次のとおりとする。

- Domain の内部で分類を記述する
- partition は分類軸を明示する
- Others は補集合としてのみ定義する
- 監査エンジンは排他性不足、被覆性不足、分類軸の揺れを評価する
- 監査結果は mece_assessment として返す

## Alternatives Considered

- MECE を厳密な論理証明対象にする
  - 形式的には強いが、主目的から外れ、実装と運用が重くなりすぎるため不採用
- MECE を扱わない
  - 分類記述の規律が弱くなり、思考監査の価値が下がるため不採用

## Consequences

- MECE 検査は記述の妥当性評価として位置付く
- Others の使用には Domain と axis の明示が必須になる
- mece_assessment は warning になりやすいカテゴリになる
- 将来、厳密論理へ拡張する場合も、MVP の前提は崩さないようにする必要がある

## Auditability Notes

- MECE を厳密証明へ拡張する提案が出た場合に再評価する
- Others を自由ラベルとして許可したくなった場合に再評価する
- partition の実運用で記述負荷が高すぎると判明した場合に再評価する
